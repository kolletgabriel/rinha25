# Rinha de Backend 2025

**TL;DR:** a playful exercise on managing 3rd party services downtime by implementing a failover mechanism under the active-passive strategy.

- Your client wants to pay you;
- you need to forward the payment to a gateway;
- the gateway might be unavailable, but you have a fallback one;
- the fallback one charges more per payment;
- and it can go unavailable too.

Instructions on how to implement it yourself are found [here](./INSTRUCTIONS.md).

## A Brief Introduction

First and foremost, the word "rinha" is a brazilian slang for "brawl", "cockfight", "cage match", "rumble" *etc*. Thus, "Rinha de Backend" means something as "cockfighting but between web services".

It's a challenge started in 2023, made for/by the dev community in Brazil, which consists in delivering a RESTful service easy to implement yet tricky to make it survive the load tests. People submit their implementations as PRs on the official repositories, and each edition presents a particular problem to be solved, with some winning criteria.

Students and seasoned developers are welcome to participate. The main purpose is to share knowledge and, of course, have fun!

This repository implements the 3rd edition of the challenge. The [official repository](https://github.com/zanfranceschi/rinha-de-backend-2025) containing the original instructions, submissions, scoreboard, source code for the tests *etc* has **everything written in pt-br** for those who prefer.
